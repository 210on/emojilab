# デフォルトカラーセットの選定根拠

更新日: 2026-07-15  
対象: EmojiLab. 標準色 `PCCS bright tone b2-b24` 12色

## 1. 結論

標準12色にPCCS bright toneを採用する主な理由は、**色相を広く、規則的かつ再現可能に提示できること**である。PCCSは色をHueとToneで系統化するため、同じトーンカテゴリのまま色相条件を変える研究刺激を定義しやすい。

一方、この12色を**等知覚間隔、等コントラスト、アクセシビリティ適合色、または絵文字に最適な色**とは扱わない。実測では背景適合性が色相によって大きく異なるため、EmojiLab.は色選択とコントラスト評価を分離し、必要な場合だけ内側線・外側線を提案する。

論文では次の記述が妥当である。

> 本研究では、任意選択による色相の偏りを抑え、色相条件を再現可能に提示するため、Hue-Tone SystemであるPCCSのbright toneから12色相を初期パレットとして採用した。この選択は各色の可読性が等しいことを意味しないため、背景および隣接線との知覚コントラストをAPCAで別途評価した。

## 2. 選定理由と根拠の強さ

| 主張 | 位置づけ | 根拠 |
|---|---|---|
| PCCSは色相とトーンを系統的に扱える | 外部根拠あり | 日本色彩研究所はPCCSをHueとToneの2系列で構成するHue-Tone Systemと説明している |
| 12色相を同じトーンカテゴリから取ると条件を再現しやすい | 方法論上の帰結 | 恣意的な色名選択ではなく、色相番号とトーン記号で刺激を再生成できる |
| 色相を変えたときの傾向を観察しやすい | 外部研究と整合 | Wakata & SaitoはPCCSの12色相と11トーンを用いて、色相とトーンの印象構造を分けて検討した |
| bright toneは絵文字に最適である | 未検証 | 現段階では制作UIの操作的選択。Study 1/2で選好・識別・制作結果を検証する必要がある |
| bright toneはvivid toneより眩しくない | 現段階では主張しない | 主観評価または表示輝度を統制した比較実験がないため |
| 色の表現意図を残す価値がある | 間接的根拠 + 設計推論 | 色選好が対象物への感情的連合と関係するという実証研究はあるが、カスタム絵文字への適用は別途検証が必要 |

### 観察

- 白背景のAPCA Lcは`8-76`、黒背景では`25-97`であり、同一トーンでも背景適合性は等しくない
- 白背景で現行の良好帯`Lc >= 75`に入るのはb18とb20、許容帯`60-74`は5色、`60`未満は5色
- WCAG 2.2 AAの通常テキスト基準`4.5:1`を単色のまま白背景で満たすのはb18、b20、b22の3色、黒背景では10色
- b22だけが白・黒の双方で`4.5:1`以上だが、この結果だけで16pxカスタム絵文字の判読性を保証することはできない
- 現行UIは色名を直接条件にせず、白背景Lc、OKLCH明度・彩度、現在の内側線効果から黒内側線を推奨する。その結果としてb6、b8、b10が該当する

## 4. なぜコントラスト均一パレットにしないのか

すべての色を白背景で同じ高コントラストにすると、黄や黄緑は大きく暗くする必要があり、初期選択肢の色相表現が狭くなる。逆に同一明度の色空間パレットは背景差を揃えやすいが、彩度や色名らしさが均一になるとは限らない。

EmojiLab.では次の役割分担を採る。

1. **初期パレット**: 色相表現を系統的に探索する出発点
2. **APCA評価**: 現在の塗り、隣接線、ライト・ダーク背景の実際の分離を評価
3. **縮小耐性**: 線を追加したことで16-20px付近の字面が詰まるリスクを評価
4. **改善コメント**: 色を一律に暗くするのではなく、必要最小限の線追加または色変更を提案

W3CのG18も、背景との比率を確保する手段として細い黒輪郭やhaloを例示している。ただし、EmojiLab.のAPCA閾値と線幅はWCAG適合判定ではなく、カスタム絵文字制作のための操作的ヒューリスティックである。

## 5. 妥当な主張と避ける主張

### 現時点で主張できる

- 色体系に基づく12色相を使うことで、初期色の選択を再現可能にした
- 標準色ごとの背景コントラストを実測し、色名ではなく数値と線の有効性から支援の要否を判定している
- PCCSを色相探索、APCAを背景適合、縮小耐性を線・字面密度の評価に使い、異なる役割を混同していない

### 人間実験前には主張しない

- bright toneがカスタム絵文字に最適である
- 12色が知覚的に等間隔、同程度に目立つ、または同程度に読みやすい
- APCA LcまたはWCAG比だけで16px絵文字の認識精度を保証できる
- 黒内側線の推奨がすべてのフォント、文字、背景で最適である

## 6. 検証計画

Study 1では、色相、背景、内側線の有無を層化し、16px、20px、32pxで文字転記正答率、反応時間、7件法の読みやすさを測る。bright tone採用の妥当性は、次の仮説へ分解する。

- H1: APCA Lcが低い条件ほど転記正答率が低く、反応時間が長い
- H2: 低Lcの明色では、有効な黒内側線が無線条件より正答率を改善する
- H3: 直接コントラストが十分な色では、不要な内側線が高画数文字の正答率を改善せず、場合によって低下させる
- H4: bright tone 12色の選好・色らしさ評価には色相差が残り、コントラストだけでは説明できない

H1-H3は視認性支援ロジック、H4は色相表現を残す設計判断を検証する。これにより「可読性と表現意図の両立」を主観的な一文ではなく、認識成績と色評価という別の従属変数で扱える。
