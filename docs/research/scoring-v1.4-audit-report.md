# Scoring v1.4 Audit Report

Date: 2026-07-15  
Metric: **metric-rules-v1.4.0**

## 1. 変更目的

v1.3は上段・下段を連結した総文字数で減点していたため、`2文字 + 2文字`にも4文字分の減点`16`が入っていた。2行化によって1行あたりの混雑を解消しているにもかかわらず、文字量警告が改善コメントとして選ばれる場合があった。

v1.4では文字量を**最も混雑した1行**で評価する。これにより、表示面積を実際に圧迫する「1行への集中」と、総文字数を分離する。

## 2. 現行ルール

| 最も混雑した行 | 文字数減点 | 単独時の縮小耐性帯 |
|---|---:|---|
| 1〜2文字 | 0 | 緑 |
| 3文字 | 19 | 黄 |
| 3文字、すべてひらがな・カタカナ | 0 | 緑（初期例外） |
| 4文字 | 29 | 赤 |
| 5文字以上 | 32 | 赤 |

上段と下段の減点は加算せず、大きい方だけを採る。`2 + 2`は0、`3 + 1`は19、`4 + 0`は29になる。

かな3文字の例外と各点数は、外部規格ではなくEmojiLab.の初期ヒューリスティックである。「かなは必ず読みやすい」という主張ではなく、Study 1で文字種別の正答率・反応時間を比較するための検証対象として扱う。

## 3. 改善コメント

- 1段で3文字以上: 上下2段化または短い・単純な表現への変更を提案
- すでに2段で片方が3文字以上: 上下への文字数再配分または表現の短文化を提案
- `2 + 2`: 文字量候補を生成しない。他の実在する減点原因だけを指摘
- かな3文字: 文字量候補を生成しない。ただし画数、線幅、変形など別原因は通常どおり評価

## 4. 自動監査

~~~bash
npm run audit:score:export
~~~

監査には次の境界ケースを含む。

- `AB / CD`: 文字数減点0、文字量警告なし
- `ありが` 1段: かな3文字例外で文字数減点0
- `ABC` 1段: 文字数減点19、縮小耐性は黄色帯
- `ありがと` 1段: 文字数減点29、縮小耐性は赤帯、2段化を提案
- `ABC / D`: 文字数減点19、既存2段への再分割ではなく行間再配分を提案
- ZWJ絵文字と結合文字: Unicodeグラフェム単位で計数

1,125条件の組合せ監査と個別境界ケースが通過した場合だけ、[`data/scoring-audit-v1.4.csv`](./data/scoring-audit-v1.4.csv)を生成する。

## 5. 妥当性と限界

今回の変更は、同じ総文字数でも`4 + 0`より`2 + 2`の方が1行あたりの文字面積を確保できるという幾何学的整合性を改善する。一方、文字数だけで実際の字幅、フォント、画数、認識難度を完全には表せない。

外的妥当性の確認には、16px、20px、32pxで以下を測定する。

- 1行2 / 3 / 4文字の文字転記正答率
- 同じ4文字を`4 + 0`と`2 + 2`にした場合の正答率・反応時間
- かな3文字、Latin 3文字、低画数漢字3文字、高画数漢字3文字の差
- 現行の緑・黄・赤境界と人間評価分布の対応

実験前に「3文字19点」「4文字29点」「かな3文字例外」を普遍的な可読性定数として主張してはならない。
