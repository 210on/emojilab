EmojiLab. Research Note

デザインスコアの仕組み

UI上の実例から、APCAコントラスト、縮小耐性、構成安定性、改善コメントまでを一段ずつ分解する。

0. 前提

デザインスコアは、カスタム絵文字の「良し悪し」を完全に決める絶対評価ではない。目的は、制作中のユーザーに対して、見やすさを損ないやすい条件を即時に返すことである。

論文では「APCAコントラスト、縮小耐性、構成安定性を統合したルールベースの設計支援指標」と説明する。人間の視認性を直接測る標準尺度ではなく、人間評価との対応を検証する対象である。

WCAG 2.2 はアクセシビリティ評価をテスト可能な success criteria として定義するが、カスタム絵文字のような画像化された小サイズ文字の美的・文脈的評価までは直接扱わない。そのため本研究では、標準指標を参照しつつ、制作支援用の操作的指標として設計する。

基準の読み方。 このページの閾値は、(1) WCAG/APCA/Unicode/Slack/Discord など外部仕様に基づく部分、(2) 現システムの実装上の操作的定義、(3) Study 1 / Study 2 で検証する初期ヒューリスティック、の3種類に分けて扱う。

黄色そのものは白背景に弱い。しかし、黒い内側線が塗り色との局所コントラストを作り、白い外側線がダーク背景上で輪郭を作るため、総合的には高評価になる。

WCAG 2.2 の Contrast Minimum は通常テキストや大きなテキストの比率基準を定める。一方、絵文字では塗り・線・背景の多層構造になるため、単一の文字色/背景色ペアだけではなく、局所コントラストと背景分離を分けて評価する。

3. スコアを8ステップで分解する

1

色と線を層として分ける

文字絵文字は「塗り vs 背景」だけでは評価できない。塗り、内側線、外側線、背景の関係を分けて見る。

内側線
外側線
背景

Slack は透明背景の正方形画像を推奨しており、Discord もカスタム絵文字をアップロード画像として扱う。画像文字では、文字本体だけでなく縁取りと透明背景を含む合成結果が実際の視対象になる。

2

有効な線だけを評価する

線が細すぎる場合、APCA値が高くても実際には境界として機能しにくい。そのため、内側線は2以上、外側線は4以上で有効と扱う。

これは外部標準の固定値ではなく、現行キャンバス設定に対する操作的定義。小サイズ表示では細線がアンチエイリアスや縮小で失われるため、色差だけで線の効果を過大評価しないための保守的な判定である。

3

APCAの代表値を作る

UIの Lc 80 は、単純な塗り色と背景色の値ではない。局所コントラストと背景分離のうち、弱い方を代表値として表示する。

displayedContrastLc =
  min(localTextLc, backgroundSeparationLc)

APCA は、従来の単純な比率よりも知覚コントラスト寄りに設計されたコントラスト手法として参照している。本システムでは APCA の Lc を直接「絶対的な合否」にはせず、多層絵文字用に代表値へ集約する。

4

総合点用の内部スコアに変換する

APCA の参照値は displayedContrastLc として UI に残す。一方、総合スコアでは縮小耐性や構成安定性と合成する必要があるため、localTextLc と backgroundSeparationLc を 0-100 の内部点 contrastFitScore に変換する。

contrastFitScore =
  normalizeLc(localTextLc) * 0.55
  + normalizeLc(backgroundSeparationLc) * 0.45

この値はAPCAそのものではなく、総合支援スコア用の派生指標である。論文やUIでAPCAとして参照する値は displayedContrastLc / Lc とし、contrastFitScore は内部計算上のコントラスト適合点として区別する。

5

縮小耐性を見る

文字数、画数、ウェイト、内側線、外側線、横幅補正、行間、字間から、小サイズで潰れやすいかを推定する。

漢字画数は Unicode Unihan Database の kTotalStrokes を用いる。画数は視認性そのものではないが、字面密度を近似する再現可能な特徴量として採用している。

6

構成安定性を見る

正方形領域を効率よく使えているか、上下行の幅が極端にずれていないか、圧縮・行間が過剰でないかを見る。

Slack は正方形画像を推奨しているため、正方形領域の利用効率は実利用上の妥当な観点である。ただし正方形性は意味や表現意図より優先しないため、総合スコアでは20%の補助指標に留める。

7

総合スコアに統合する

designScore =
  contrastFitScore * 0.45
  + scalabilityScore * 0.35
  + compositionScore * 0.20

45/35/20 の重みは標準規格ではなく、カスタム絵文字の制作支援における重要度の初期仮説。コントラストを基礎、縮小耐性を小サイズ表示への適合、構成安定性を補助として扱う。

8

改善コメントに変換する

すべての指標を列挙せず、次に直すべき一点だけを返す。固定順序ではなく、コントラスト不足、背景分離不足、高画数漢字、内側線過多、外側線不足、縦横比の偏りなどの候補を並べ、推定改善インパクトが最も大きいものを選ぶ。

comment =
  maxImpact(
    contrast issue,
    scalability issue,
    stroke issue,
    composition issue
  )

UI上の認知負荷を下げるため、詳細指標を増やすのではなく、総合スコアと短い改善コメントに統合する。妥当性は、コメント後の操作ログで実際にその問題が修正されるか、成果物評価が上がるかで検証する。

6. 研究上の限界

このスコアは、視線移動、個人差、表示環境、意味理解、使用意欲を直接測っていない。研究では、第三者評価、文字転記課題、制作支援比較実験によって妥当性を検証する。

スコアは「完全な正解」ではなく、制作中に見やすさのリスクを返すための操作的指標である。
