# EmojiLab Research Experiment Suite

## 目的

EmojiLab本体の制作UIとは別に、カスタム絵文字制作支援が人間の評価・制作行動・成果物品質に与える影響を検証するための研究用システムを追加する。

本研究用システムは、制作支援機能そのものを作ることではなく、制作支援の効果を検証可能にすることを目的とする。

## 現在の実装範囲

- `/research/admin`: 研究用入口、刺激セットと指標バージョン確認
- `/research/study1`: 視認性評価実験の最小フロー
- `/research/export`: localStorageに保存された研究データのJSON / CSV出力
- `/research/debug`: 実験前チェック
- `src/research/metrics`: APCA、縮小耐性、構成評価、総合支援スコアの決定論的計算
- `src/research/storage`: ブラウザ内保存、CSV / JSON / ダウンロード処理

## 実装方針

初期実装では外部DBを使わない。各回答・イベントはブラウザ内に即時保存し、研究者がJSON / CSVとして回収する。

この方針は、倫理審査前後で収集項目が変化しうる段階でも扱いやすく、ネットワーク障害時にもデータ喪失を抑えられる。

## 研究上の注意

デザインスコアは、客観的な視認性尺度ではなく、APCAコントラスト・縮小耐性・文字構成にもとづくルールベースの設計支援指標である。

論文では、以下の表現を用いる。

- 設計支援指標
- ルールベース評価
- 暫定的ヒューリスティック
- 人間評価との対応を検証する対象

以下の表現は避ける。

- 客観的視認性スコア
- 正確な読みやすさ指標
- 人間の知覚を直接測定する尺度
